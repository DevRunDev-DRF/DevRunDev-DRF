# DevRunDev 온라인 강의 플랫폼 - 아키텍처 설계 문서 (ARD)

## 1. Django App 설계
| **App 명** | **설명** | **다루는 Model** |
|------------|---------|----------------|
| **accounts** | 사용자 관리, 권한 관리, 인증 및 강사 신청 관리 | `User`, `InstructorApplication` |
| **courses** | 강의 및 강의 콘텐츠(섹션, 레슨) 관리 | `Course`, `Section`, `Lesson` |
| **enrollments** | 수강 신청, 학습 진행 및 결제 관리 | `Enrollment`, `LessonProgress`, `Certificate`, `CartItem`, `Payment` |
| **quizzes** | 퀴즈, 문제 및 퀴즈 시도 관리 | `Quiz`, `Question`, `Choice`, `QuizAttempt`, `Answer` |
| **reviews** | 강의 리뷰 관리 | `Review` |
| **qna** | 강의 질문 및 답변 관리 | `Question`, `Answer` |
| **config** | 프로젝트 설정 및 관리자 대시보드 | 없음 (관리자 뷰 포함) |

## 2. 데이터베이스 모델 설계

### accounts App
- **User Model**
  - 상속: `AbstractUser`
  - 필드:
    - `email`: 이메일 (기본 로그인 ID)
    - `username`: 사용자명
    - `role`: 역할 (학생, 강사, 관리자)
    - `is_verified`: 사용자 이메일 인증 여부
  - 관계:
    - `instructor_applications`: 강사 신청 목록 (역참조)
    - `course_set`: 강사가 만든 강의 목록 (역참조)
    - `enrollments`: 수강 신청 목록 (역참조)
    - `quiz_attempts`: 퀴즈 시도 목록 (역참조)
    - `reviews`: 작성한 리뷰 목록 (역참조)

- **InstructorApplication Model**
  - 필드:
    - `user`: 신청자 (FK)
    - `qualifications`: 자격 사항
    - `experience`: 경력 사항
    - `sample_video_url`: 샘플 강의 영상 URL
    - `status`: 신청 상태 (대기, 승인, 거부)
    - `created_at`: 신청일
    - `updated_at`: 수정일

### courses App
- **Course Model**
  - 필드:
    - `instructor`: 강사 (FK)
    - `title`: 강의 제목
    - `description`: 강의 설명
    - `price`: 강의 가격
    - `status`: 강의 상태 (심사 중, 승인됨, 수정 필요)
    - `thumbnail`: 강의 썸네일 이미지
    - `created_at`: 생성일
    - `updated_at`: 수정일
    - `avg_rating`: 평균 평점
  - 관계:
    - `sections`: 섹션 목록 (역참조)
    - `enrollments`: 수강 신청 목록 (역참조)
    - `reviews`: 리뷰 목록 (역참조)
    - `quizzes`: 퀴즈 목록 (역참조)
    - `questions`: Q&A 질문 목록 (역참조)

- **Section Model**
  - 필드:
    - `course`: 강의 (FK)
    - `title`: 섹션 제목
    - `order`: 정렬 순서
  - 관계:
    - `lessons`: 레슨 목록 (역참조)
    - `quizzes`: 퀴즈 목록 (역참조)

- **Lesson Model**
  - 필드:
    - `section`: 섹션 (FK)
    - `title`: 레슨 제목
    - `video_url`: 동영상 URL (YouTube)
    - `order`: 정렬 순서
  - 관계:
    - `quizzes`: 퀴즈 목록 (역참조)
    - `progress`: 학습 진행 상황 (역참조)
    - `questions`: Q&A 질문 목록 (역참조)

### enrollments App
- **Enrollment Model**
  - 필드:
    - `student`: 학생 (FK)
    - `course`: 강의 (FK)
    - `status`: 상태 (수강 중, 완료, 수강 취소)
    - `progress`: 진행률
    - `enrolled_at`: 수강 신청일
    - `last_updated_at`: 최종 업데이트일
    - `last_watched_lesson`: 마지막 시청 레슨 (FK)
  - 관계:
    - `certificate`: 수료증 (일대일)

- **LessonProgress Model**
  - 필드:
    - `student`: 학생 (FK)
    - `lesson`: 레슨 (FK)
    - `completed`: 완료 여부
    - `completed_at`: 완료일
    - `last_watched_at`: 마지막 시청일

- **Certificate Model**
  - 필드:
    - `enrollment`: 수강 신청 (FK)
    - `certificate_id`: 수료증 ID
    - `issued_at`: 발급일

- **CartItem Model**
  - 필드:
    - `user`: 사용자 (FK)
    - `course`: 강의 (FK)
    - `added_at`: 추가일

- **Payment Model**
  - 필드:
    - `user`: 결제자 (FK)
    - `merchant_uid`: 주문번호
    - `imp_uid`: 아임포트 결제 고유번호
    - `amount`: 결제 금액
    - `status`: 결제 상태 (대기, 완료, 취소, 실패)
    - `created_at`: 생성일
    - `updated_at`: 수정일
  - 관계:
    - `cart_items`: 장바구니 항목 (M2M)

### quizzes App
- **Quiz Model**
  - 필드:
    - `title`: 퀴즈 제목
    - `description`: 퀴즈 설명
    - `course`: 강의 (FK)
    - `section`: 섹션 (FK, Optional)
    - `lesson`: 레슨 (FK, Optional)
    - `instructor`: 출제자 (FK)
    - `created_at`: 생성일
    - `updated_at`: 수정일
  - 관계:
    - `questions`: 문제 목록 (역참조)
    - `attempts`: 퀴즈 시도 목록 (역참조)

- **Question Model**
  - 필드:
    - `quiz`: 퀴즈 (FK)
    - `text`: 문제 내용
    - `order`: 정렬 순서
  - 관계:
    - `choices`: 선택지 목록 (역참조)
    - `answers`: 답변 목록 (역참조)

- **Choice Model**
  - 필드:
    - `question`: 문제 (FK)
    - `text`: 선택지 내용
    - `is_correct`: 정답 여부
  - 관계:
    - `selections`: 선택된 답변 목록 (역참조)

- **QuizAttempt Model**
  - 필드:
    - `quiz`: 퀴즈 (FK)
    - `student`: 학생 (FK)
    - `started_at`: 시작 시간
    - `completed_at`: 완료 시간
    - `score`: 점수
    - `total_questions`: 전체 문제 수
    - `correct_answers`: 정답 수
    - `is_completed`: 완료 여부
  - 관계:
    - `answers`: 답변 목록 (역참조)

- **Answer Model**
  - 필드:
    - `attempt`: 퀴즈 시도 (FK)
    - `question`: 문제 (FK)
    - `selected_choice`: 선택한 답 (FK)
    - `is_correct`: 정답 여부

### reviews App
- **Review Model**
  - 필드:
    - `user`: 작성자 (FK)
    - `course`: 강의 (FK)
    - `rating`: 평점 (1-5)
    - `comment`: 리뷰 내용
    - `created_at`: 작성일
    - `updated_at`: 수정일

### qna App
- **Question Model** (Q&A용)
  - 필드:
    - `user`: 작성자 (FK)
    - `course`: 강의 (FK)
    - `lesson`: 레슨 (FK, Optional)
    - `title`: 질문 제목
    - `content`: 질문 내용
    - `is_resolved`: 해결 여부
    - `created_at`: 작성일
    - `updated_at`: 수정일
  - 관계:
    - `answers`: 답변 목록 (역참조)

- **Answer Model** (Q&A용)
  - 필드:
    - `question`: 질문 (FK)
    - `user`: 작성자 (FK)
    - `content`: 답변 내용
    - `is_accepted`: 채택 여부
    - `created_at`: 작성일
    - `updated_at`: 수정일

## 4. 권한 설계

### 사용자 역할
- **학생 (Student)**
  - 강의 조회
  - 강의 수강 신청
  - 퀴즈 응시
  - 리뷰 작성 (수강 중인 강의만)
  - Q&A 질문 및 답변 작성
  - 자신의 학습 진행 상황 조회

- **강사 (Instructor)**
  - 학생 권한 모두 포함
  - 강의 생성 및 관리
  - 강의 콘텐츠 관리 (섹션, 레슨)
  - 퀴즈 생성 및 관리
  - 자신의 강의에 대한 질문 답변
  - 자신의 강의 수강생 관리

- **관리자 (Manager)**
  - 모든 권한 포함
  - 사용자 관리
  - 강사 신청 관리
  - 모든 강의 관리 및 승인
  - 결제 관리
  - 시스템 설정 관리

### 권한 클래스
- **IsInstructor**: 강사 권한 확인
- **IsManager**: 관리자 권한 확인
- **IsStaffOrManager**: 관리자 또는 스태프 권한 확인
- **IsInstructorOrReadOnly**: 강사만 생성/수정 가능, 다른 사용자는 조회만 가능
- **IsOwnerInstructorOrReadOnly**: 강의 생성자(강사)만 수정 가능, 다른 사용자는 조회만 가능
- **IsEnrollmentOwner**: 수강신청 소유자(학생)만 자신의 수강 정보 접근 가능
- **IsCartItemOwner**: 장바구니 아이템 소유자만 접근 가능
- **IsQuizInstructor**: 퀴즈 생성자(강사)만 수정 가능
- **IsEnrolledOrInstructor**: 수강 중인 학생이나 강사만 퀴즈 접근 가능
- **IsQuizAttemptOwner**: 퀴즈 시도 소유자(학생)만 접근 가능
- **IsQuestionOwnerOrReadOnly**: 질문 소유자만 수정/삭제 가능, 다른 사용자는 조회만 가능
- **IsAnswerOwnerOrReadOnly**: 답변 소유자만 수정/삭제 가능, 다른 사용자는 조회만 가능
- **CanAcceptAnswer**: 질문 작성자 또는 강의 강사만 답변 채택 가능
- **IsReviewOwnerOrReadOnly**: 리뷰 소유자만 수정/삭제 가능, 다른 사용자는 조회만 가능

## 5. API 설계

### REST API 엔드포인트
- 모든 주요 모델에 대해 CRUD 작업을 수행할 수 있는 RESTful API 제공
- Swagger 및 ReDoc을 통한 API 문서화
- Django REST Framework의 ViewSet을 활용한 표준 API 구현
- 필터링, 검색, 정렬 기능 지원
- 토큰 기반 인증 구현

### 주요 API 기능
- **인증 API**: 로그인, 로그아웃, 회원가입
- **사용자 API**: 사용자 정보 CRUD
- **강의 API**: 강의 및 콘텐츠 CRUD
- **수강 API**: 수강 신청, 진행 상황 관리
- **퀴즈 API**: 퀴즈 생성, 응시, 결과 조회
- **리뷰 API**: 리뷰 CRUD
- **Q&A API**: 질문 및 답변 CRUD
- **결제 API**: 결제 처리 및 관리
